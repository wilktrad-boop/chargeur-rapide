## Qu'est-ce que Rybbit ?

Rybbit est un **outil d'analytics web open source** qui se positionne comme alternative légère et respectueuse de la vie privée à Google Analytics. Créé par un développeur indépendant, il a explosé sur Reddit et ProductHunt en 2024-2025 avant de rejoindre AppSumo pour une offre lifetime deal.

Ses caractéristiques principales :

- **Cookieless** : aucun cookie de suivi, conformité RGPD nativement
- **Temps réel** : tableau de bord mis à jour en direct
- **Replays de sessions** : voir exactement comment les visiteurs naviguent
- **Open source** : code disponible sur GitHub, self-hostable ou cloud
- **Script léger** : < 2 KB, n'impacte pas les performances

## Fonctionnalités principales de Rybbit

### Tableau de bord en temps réel

L'interface de Rybbit est volontairement épurée. En un coup d'œil : visiteurs actifs en ce moment, pages vues, sources de trafic, pays, appareils. Tout s'actualise en temps réel sans délai de 24-48h comme sur GA4.

### Replays de sessions

C'est la fonctionnalité qui fait la différence. Rybbit enregistre les sessions de vos visiteurs (avec anonymisation des données personnelles) et vous permet de les rejouer : mouvements de souris, clics, scroll. Idéal pour comprendre pourquoi les utilisateurs abandonnent un formulaire ou ne trouvent pas un bouton.

### Suivi d'événements personnalisés

Au-delà des pages vues, vous pouvez tracker des événements spécifiques : clics sur un CTA, soumissions de formulaire, téléchargements, conversions. L'implémentation est simple (une ligne de JavaScript).

### Carte géographique et données démographiques

Répartition des visiteurs par pays et ville, avec carte interactive. Données d'appareils, OS, navigateurs. Le tout sans stocker d'IP complète — conformité RGPD préservée.

### Suivi sans cookies (cookieless tracking)

Rybbit identifie les visiteurs via un hash anonymisé qui ne persiste pas entre les sessions. Résultat : aucun bandeau cookie nécessaire, aucune donnée personnelle stockée. Pour les sites soumis au RGPD (tous les sites européens), c'est un avantage considérable.

## Installation et prise en main

### Option 1 : Cloud (le plus simple)

1. Créer un compte sur rybbit.com
2. Ajouter votre site (renseigner le domaine)
3. Copier le script de tracking (une ligne de code)
4. Coller dans le `<head>` de votre site
5. Données disponibles en temps réel

**Temps total : 5 minutes.** Aucune configuration serveur.

### Option 2 : Self-hosted (pour les développeurs)

Rybbit utilise **ClickHouse** comme base de données (nécessaire pour les performances analytiques à grande échelle). Le déploiement se fait via Docker Compose.

```
git clone https://github.com/rybbit-io/rybbit
cd rybbit
docker compose up -d
```

Des guides existent pour déployer sur **Coolify**, Railway et Hetzner. La complexité principale est la configuration de ClickHouse — plus engageant qu'un simple `docker run`.

<Callout type="warn">
Si vous n'êtes pas développeur, optez pour la version cloud. Le self-hosting de Rybbit nécessite des connaissances en administration de serveurs et en Docker.
</Callout>

## Limites et inconvénients à connaître

### ClickHouse = self-hosting plus exigeant

Rybbit utilise ClickHouse (une base de données analytique orientée colonne) pour les performances. C'est une excellente technologie, mais elle ajoute de la complexité au self-hosting comparé à Umami (PostgreSQL ou MySQL simple).

### Produit jeune

Rybbit est sorti en 2024. Malgré une croissance rapide, il manque encore de maturité sur certains aspects : rapports d'entonnoir (funnels), A/B testing, intégrations natives CRM. Ces fonctionnalités sont sur la roadmap.

### Pas de plugin WordPress officiel

L'intégration WordPress se fait manuellement (ajout du script dans le header). Un plugin officiel est attendu mais pas encore disponible en 2026.
